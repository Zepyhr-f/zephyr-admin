## 一、核心基础 
Spring Cloud Gateway 核心由 Route（路由）、Predicate（断言） 和 Filter（过滤器） 组成。

配置
```yaml
spring:
    cloud:
      gateway:
        routes:
          - id: cloud-order-service
            uri: lb://cloud-order-service
            predicates:
              - Path=/order/**
            filter:
              - 
```

### 1.1 路由（Route）
路由就是一条转发规则，由一个ID，一个目标URI，一组断言和过滤器组成。其中断言决定请求是否命中路由，过滤器对请求进行增强处理。

### 1.2 断言（Predicate）
断言用于实现请求匹配逻辑，例如匹配路径、请求头、请求参数等。

**常用断言工厂**
- Path：
  - ?：匹配单个字符
  - *：匹配任意字符（不含 /）
  - **：匹配任意路径（含 /）
- Method：请求方法匹配
- Query：查询参数匹配
- Host：域名匹配
- Header：请求头匹配
- After/Before/Between：时间匹配

**自定义断言**
- 名称 <yourPredicateName>RoutePredicateFactory
- 继承AbstractRoutePredicateFactory
- 编写配置类config，定义参数（字段名、阈值）。
- 实现 apply(Config) 方法

### 1.3 过滤器（Filter）
Filter 用于对请求/响应进行增强处理

**过滤器分类**
- 全局过滤器：所有路由都生效
- 局部过滤器：只生效指定路由

**常见内置过滤器**
- AddRequestHeader：添加请求头
- AddResponseHeader：添加响应头
- RewritePath：重写路径
- StripPrefix：去掉路径前缀
- RequestRateLimiter：基于 Redis 的限流
- Retry：失败重试
- CircuitBreaker：熔断

**自定义过滤器**
- 继承GatewayFilter自定义局部过滤器
- 继承GlobalFilter自定义全局过滤器
