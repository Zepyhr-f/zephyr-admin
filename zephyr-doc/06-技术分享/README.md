## 阅读建议
- 当主题是“某个技术点怎么落地”或“为什么这样配”时，优先来本目录找补充说明。
- 如果问题属于长期稳定的架构边界或统一规范，应优先回到架构设计或开发规范目录。

## 修改指引
- 当在开发中遇到非通用但值得记录的技术方案（如踩坑解决方案、特定中间件的最佳实践）时，请在此创建新文档。
- 内容格式建议：**问题背景 → 分析过程 → 最终方案 → 注意事项**。
