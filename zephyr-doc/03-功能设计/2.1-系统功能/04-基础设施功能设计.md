# 基础设施功能详细设计

本文档详细描述了 Zephyr 基础设施模块的具体实现，包括字典、参数、文件及通知公告等支撑系统运行的核心功能。

## 1. 字典管理 (Dict Management)
维护系统运行所需的各类静态数据（如：性别、状态类型、任务分类等）。

- **涉及表**：`sys_dict_type` (字典类型), `sys_dict_data` (字典数据)
- **查询条件**：字典名称、字典类型、状态。
- **功能逻辑**：
  - **字典类型**：作为分类容器（如 `sys_user_sex`），支持全局唯一的类型编码。
  - **字典数据**：具体的键值对（如 `1: 男`, `2: 女`）。
- **列表展示字段**：
  - `dict_name`: 字典名称
  - `dict_type`: 字典类型 (编码)
  - `status`: 状态 (正常/停用)
  - `remark`: 备注
- **核心按钮**：
  - `列表管理`: 跳转至明细数据管理页。
  - `刷新缓存`: 将字典数据同步至 Redis，供后端 `DictUtils` 高速读取。

## 2. 参数配置 (Config Management)
动态管理系统运行期间的全局参数，支持热更新（无需重启服务）。

- **涉及表**：`sys_config`
- **查询条件**：参数名称、参数键名、是否内置。
- **列表展示字段**：
  - `config_name`: 参数名称
  - `config_key`: 参数键名 (如 `sys.user.initPassword`)
  - `config_value`: 参数键值
  - `config_type`: 系统内置 (Y/N)
- **应用场景**：
  - 系统默认密码、侧边栏风格外观、文件上传大小限制等。
- **功能维护**：
  - 修改后立即更新 Redis 缓存，并广播至各微服务节点。

## 3. 文件管理 (File Management)
统一管理系统内所有上传的静态资源文件。

- **涉及表**：`sys_file` (文件记录)
- **展示形式**：卡片或列表，支持预览。
- **核心字段**：
  - `file_name`: 原始文件名
  - `file_url`: 访问地址 (CDN 或 代理地址)
  - `file_size`: 文件大小
  - `file_type`: 扩展名 (png/jpg/pdf等)
  - `store_type`: 存储位置 (local/oss/minio)
- **功能按钮**：
  - `上传`: 支持大文件切片上传与秒传（基于 MD5 校验）。
  - `下载`: 直接流式下载。
  - `删除`: 同时清理数据库记录与物理磁盘/对象存储。

## 4. 通知公告 (Notice Management)
发布系统全局消息或定向推送。

- **涉及表**：`sys_notice`
- **核心字段**：
  - `notice_title`: 公告标题
  - `notice_type`: 类型 (通知/公告)
  - `notice_content`: 内容 (支持富文本编辑器)
  - `status`: 发布状态 (草稿/已发布)
- **展示形式**：
  - 顶部消息通知铃铛展示最新已发布公告。
  - 管理后台提供完整的富文本管理面板。

---

## 5. 交互与技术架构规范
1. **统一缓存机制**：字典与参数必须具备 Redis 缓存层，禁止在业务逻辑中直接频繁查询 `sys_dict_data` 表。
2. **三方存储隔离**：文件管理模块应通过策略模式 (Strategy Pattern) 实现，后端代码仅感知 `FileService` 接口，存储后端（本存、阿里云 OSS、Minio）通过配置动态切换。
3. **富文本安全性**：通知公告的内容在保存前必须经过 HTML XSS 过滤（如使用 `Jsoup`）。
4. **状态同步**：当字典或参数被修改时，前端状态应在下一次页面刷新或特定“刷新缓存”操作后同步。
