# 开发工具功能详细设计

本文档详细描述了 Zephyr 开发工具模块的具体实现，旨在通过自动化手段提升开发效率，降低重复代码编写量，并增强后端接口的可调试性。

## 1. 代码生成 (Code Generation)
实现“表驱动开发”的核心组件，根据数据库元数据自动产出标准化代码。

- **涉及表**：内部元数据缓存（如 `gen_table`, `gen_table_column`），实际上直接读取 `information_schema`。
- **核心流程**：
  1. **导入表**：从数据库连接中解析物理表结构，读取字段名、类型、注释及索引。
  2. **配置修改**：
     - `作者`、`实体类名`、`业务名`。
     - `映射规则`：前端组件类型选择（Input, Select, TreeSelect）、是否必填、查询方式（LIKE, EQ, BETWEEN）。
  3. **预览代码**：在 Web 页面实时渲染 Controller, Service, Mapper, Entity, SQL, React 组件代码。
  4. **下载/执行**：生成 Zip 包下载，或直接将菜单 SQL 注入系统表。
- **技术选型**：Velocity 模板引擎。

## 2. 接口文档 (API Documentation)
基于 OpenAPI 3.0 规范，自动化聚合微服务接口。

- **功能点**：
  - **聚合展示**：通过 Gateway 统一反向代理各微服务（system, auth, ops）的 `/v3/api-docs`。
  - **在线调试**：支持在 UI 界面配置 Headers（如 `Authorization`），直接发起请求并检视 Response。
  - **文档导出**：支持导出离线 Markdown 或 HTML 文档。
- **技术实现**：集成 `Knife4j` UI + `SpringDoc` 依赖。
- **UI 路径**：`/api/zephyr-ops/doc.html` (受网关白名单控制)。

## 3. SQL 终端 (SQL Terminal)
为开发人员提供的轻量级在线数据维护入口。

- **权限控制**：
  - **只读模式**：开发环境全开放，生产环境仅限 `admin` 角色开启。
  - **高危操作阻断**：拦截不带 `WHERE` 条件的 `UPDATE/DELETE`。
- **核心字段**：
  - `连接信息`：默认为当前数据源。
  - `执行历史`：记录当前会话执行过的 SQL 语句。
- **展示形式**：
  - **编辑器**：集成 `Monaco Editor`，支持 SQL 语法高亮与智能提示。
  - **结果集**：动态列展示 Grid。

---

## 4. 交互规范
1. **生成预警**：执行代码生成覆盖操作前，必须进行强提示，防止本地修改被意外重置。
2. **脱敏处理**：SQL 终端中涉及用户敏感信息（如密码散列值）的列应默认显示为 `***`。
3. **接口同步**：后端接口变更后，需提供一键刷新配置按钮，以同步 OpenAPI 缓存。

---
**配置说明**：开发工具模块通常作为 `zephyr-generator` 独立服务运行，以避免侵入业务核心。
