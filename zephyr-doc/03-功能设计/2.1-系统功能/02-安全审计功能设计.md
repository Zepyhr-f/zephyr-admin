# 安全审计功能设计

本文档详细描述了 Zephyr 安全审计模块的具体实现，包括登录日志、操作日志及在线用户的实现方案。

> **关联数据库脚本**：[`4.1.2-security_audit.sql`](../../04-数据库设计/4.1-建表脚本/4.1.2-security_audit.sql)

---

## 1. 登录日志 (Login Logs)

记录用户访问系统的全过程，用于追踪非法登录尝试与安全审计。

### 1.1 涉及表：`sys_login_log`

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | `BIGINT` | 主键ID（雪花算法） |
| `username` | `VARCHAR(50)` | 登录账号 |
| `ipaddr` | `VARCHAR(128)` | 登录IP地址 |
| `login_location` | `VARCHAR(255)` | 登录地点（IP解析） |
| `browser` | `VARCHAR(50)` | 浏览器类型 |
| `os` | `VARCHAR(50)` | 操作系统 |
| `status` | `TINYINT(1)` | 登录状态（0=成功 1=失败） |
| `msg` | `VARCHAR(255)` | 提示消息（如：密码错误、账号锁定） |
| `login_time` | `DATETIME` | 登录时间 |
| `create_user` | `BIGINT` | 创建人ID |
| `create_time` | `DATETIME` | 创建时间 |
| `update_user` | `BIGINT` | 更新人ID |
| `update_time` | `DATETIME` | 更新时间 |
| `del_flag` | `TINYINT(1)` | 逻辑删除标志（0=正常 1=已删除） |

### 1.2 列表展示字段
账户名、IP地址、登录地点、浏览器、操作系统、登录状态、操作信息、登录日期。

### 1.3 功能按钮
| 按钮 | 说明 |
| :--- | :--- |
| `查询` | 支持按账号、日期范围、状态过滤 |
| `删除` | 逻辑删除选中的日志记录（del_flag=1） |
| `清空` | 一键逻辑清空所有历史日志 |
| `导出` | 将当前搜索结果导出为 Excel |

## 2. 操作日志 (Operation Logs)

记录系统中所有非 GET 类型的业务操作，用于事故追溯与行为分析。

### 2.1 涉及表：`sys_oper_log`

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | `BIGINT` | 主键ID（雪花算法） |
| `title` | `VARCHAR(50)` | 系统模块名称（如：用户管理） |
| `business_type` | `TINYINT(1)` | 业务类型（0=其它 1=新增 2=修改 3=删除 4=授权 5=导出 6=导入 7=强退 8=生成代码 9=清空数据） |
| `method` | `VARCHAR(100)` | 方法名称 |
| `request_method` | `VARCHAR(10)` | 请求方式（POST/PUT/DELETE） |
| `operator_type` | `TINYINT(1)` | 操作类别（0=其它 1=后台用户 2=手机端用户） |
| `oper_name` | `VARCHAR(50)` | 操作人员账号 |
| `dept_name` | `VARCHAR(50)` | 部门名称 |
| `oper_url` | `VARCHAR(255)` | 请求URL |
| `oper_ip` | `VARCHAR(128)` | 主机地址 |
| `oper_location` | `VARCHAR(255)` | 操作地点（IP解析） |
| `oper_param` | `VARCHAR(2000)` | 请求参数（敏感字段已脱敏） |
| `json_result` | `VARCHAR(2000)` | 返回参数 |
| `status` | `TINYINT(1)` | 操作状态（0=正常 1=异常） |
| `error_msg` | `VARCHAR(2000)` | 错误消息 |
| `oper_time` | `DATETIME` | 操作时间 |
| `cost_time` | `BIGINT` | 消耗时间（ms） |
| `create_user` | `BIGINT` | 创建人ID |
| `create_time` | `DATETIME` | 创建时间 |
| `update_user` | `BIGINT` | 更新人ID |
| `update_time` | `DATETIME` | 更新时间 |
| `del_flag` | `TINYINT(1)` | 逻辑删除标志（0=正常 1=已删除） |

### 2.2 展示形式
支持查看详细 Modal 弹窗，显示完整的请求参数与返回内容。

### 2.3 功能按钮
| 按钮 | 说明 |
| :--- | :--- |
| `查询` | 按模块名称、操作人员、状态、日期范围过滤 |
| `批量删除` | 逻辑删除选中日志（del_flag=1） |
| `导出` | 审计报告导出为 Excel |

## 3. 在线用户 (Online Users)

实时监控当前正在使用系统的活跃用户。

### 3.1 实现机制
- **不依赖数据库**，通过 Redis 中 Token 集合实时获取。
- Redis Key 规则：`zephyr:auth:token:*`

### 3.2 列表展示字段
| 字段 | 说明 |
| :--- | :--- |
| `token` | 会话编号（脱敏，仅展示前8位） |
| `username` | 登录账号 |
| `dept_name` | 部门名称 |
| `ipaddr` | 登录IP |
| `login_location` | 登录地点 |
| `browser` | 浏览器 |
| `os` | 操作系统 |
| `login_time` | 登录时间 |

### 3.3 核心功能
- `强退 (Kick out)`：从 Redis 中删除对应 Token 并清理权限缓存，强制用户下线并跳转至登录页。

---

## 4. 交互与性能规范

1. **异步写入**：日志记录必须通过 `@Async` 或消息队列处理，严禁阻塞业务主流程。
2. **容量控制**：操作日志建议仅保留最近 3-6 个月的数据，通过定时任务自动清理。
3. **敏感信息脱敏**：存储 `oper_param` 时，必须过滤 `password`、`oldPassword` 等字段。
4. **逻辑删除**：日志的"删除"与"清空"操作均为逻辑删除（`del_flag=1`），严禁物理删除（`DELETE FROM`）。
