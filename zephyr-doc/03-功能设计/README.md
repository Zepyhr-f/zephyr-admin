# 02-功能设计

本模块记录了 Zephyr 项目的全体系功能规划与详细方案设计。为了保持逻辑清晰，文档划分为“系统功能”与“业务功能”两大子模块。

## 📁 子模块说明

### [2.1-系统功能](./2.1-系统功能/)
涵盖了支撑平台运行的通用后座功能，如权限、审计、监控等。
- **[00-系统功能设计总览](./2.1-系统功能/00-系统功能设计总览.md)**: 全局功能地图与优先级规划。
- **[01-系统管理](./2.1-系统功能/01-系统管理功能设计.md)**: 核心 RBAC、组织架构及数据权限设计。
- **[02-安全审计](./2.1-系统功能/02-安全审计功能设计.md)**: 行为日志审计与会话安全控制。
- **03-系统监控**: (计划中) 服务、缓存及数据库监控指引。
- **04-基础设施**: (计划中) 字典、参数及文件管理设计。

### [2.2-业务功能](./2.2-业务功能/)
涵盖了本项目特定的垂直业务逻辑（目前为空，待业务开发时补全）。

## 修改指引
- 每个细分功能的详细设计文档应包含：核心表清单、业务流程图、前端列表及按钮定义。
- 所有的功能变更必须首先在 **00-总览** 中登记，然后再更新具体的详细设计文档。
