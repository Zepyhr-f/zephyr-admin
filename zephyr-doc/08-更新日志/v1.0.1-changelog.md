# 🚀 Zephyr 发版更新日志 - v1.0.1

### 📅 发布时间
2026年04月

### 🎯 核心变更摘要 (Highlights)
本次更新主要致力于打通前后端本地全栈调试的环境配置，排查处理了权限路由失灵与中间件网络通信异常，为接下来的纯业务代码编写铺牢了底座。

### 🧩 后端体系 (zephyr-server)
- **微服务配置修正**：修复了 `zephyr-gateway` 在 Docker 容器内部无法向外发现并在 Nacos 上注册的心跳异常问题，将非标准配置 `SERVER-ADDR` 更正为合规的 `SERVER_ADDR`。
- **脚本提效**：废弃了零迷的 `startup.sh` / `shutdown.sh`，将其重组进具备多维参数管理的 `server.sh` 当中。
- **数据恢复防线**：侦破了数据库初始化测试脚本引发的管理员脏数据问题，针对 `admin` 的无效特征码利用底座原生 BCrypt 生成模块重塑了标准加密串。

### 🖥️ 前端展现 (zephyr-web)
- **动态路由菜单修复（RBAC 同步）**：解决并根除了登入后系统菜单丢失的阻塞性 Bug；纠正了因数据库角色码 `admin` 与前端本地静态控制表 `R_ADMIN` / `R_SUPER` 检测脱节导致的显示阉割。
- **集成服务接入**：在外部接入了通过启动外壳直接监控清理及 `pm2` 化处理的 Vue DevServer 持久化支持。

### 🛠️ 工作流及架构文档 (zephyr-doc & z-manager)
- **并行版本管理模式**：拆分确立了三个底层工程（后端/前端/文档）的三轨同频部署路线，均以原生 `.git` 库独立封板。
- **全栈调度中心**：搭建项目级运维枢纽 `z-run.sh` 与新版 `project.md` 的 AI 提交流程强约定，支持未来开发中仅用对话触发多线提交推送能力。
