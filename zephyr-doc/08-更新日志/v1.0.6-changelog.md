# v1.0.6 更新日志

**发版时间**: 2026-04-16

## 后端变更 (zephyr-server)
- **系统监控核心模块上线**:
  - **硬件指标监控**: 集成 `oshi-core`，实现对 CPU、内存、JVM、操作系统及磁盘信息的实时采集与接口暴露。
  - **缓存状态监控**: 实现基于 Redis `INFO` 指令的运营数据抓取，支持内存占用、客户端连接、QPS 及命令统计分析。
  - **动态任务调度 (Quartz)**: 
    - 集成 Quartz + MyBatis-Plus，支持任务的 CRUD、状态切换（起停）、立即执行及并发控制。
    - 封装 `JobInvokeUtil` 支持全自动反射调用 Spring Bean 方法，并实现异步任务日志落库。
- **网关与安全增强**:
  - `zephyr-gateway`: 新增 `PreserveHostHeader` 过滤器，修复分布式环境下 `v1.0.5` 遗留的 Druid 监控面板内网 IP 泄漏问题。
  - `SecurityConfig`: 显式禁用 `X-Frame-Options: DENY`，解决前端 Iframe 嵌套被拦截的安全性屏蔽。
- **数据库治理**: 提供了全量菜单与角色权限同步 SQL，修复了 RBAC 菜单树不完整的问题。

## 前端变更 (zephyr-web)
- **系统监控面板全量集成**:
  - 新增“服务监控”可视化页，实时展示集群硬件负载与环境参数。
  - 新增“缓存监控”看板，集成 `@ant-design/plots` 图表展示 Redis 性能趋势。
  - 新增“定时任务”调度管理，支持任务在线编辑、执行日志查看及状态实时管控。
- **架构重构与体验优化**:
  - **模块解耦**: 修改路由 `name` 为 `monitor-sys` 等唯一标识，彻底解决与原生 Dashboard 模块的菜单键名冲突（403 报错）。
  - **路由收口**: 物理删除了 Ant Design Pro 原生的 Dashboard 演示代码，并将系统根路径 (`/`) 默认跳转至全新的“服务监控”。
- **国际化补全**: 更新了 `zh-CN` 与 `en-US` 菜单翻译，并为监控子模块配置了 `access: 'canAdmin'` 权限硬拦截。
