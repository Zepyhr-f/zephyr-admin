# Zephyr 系统更新日志 (v1.0.2)
日期: 2026-04-16

## 核心更新
- **前端架构全面升级**：
  - 移除原有的简易前端，完整迁入 **Ant Design Pro (Umi/Max)** 商业级前端框架。
  - 完成了 `request` 请求拦截器重写，支持后端 `R` 对象规范及 `Bearer Token` 自动注入。
  - 适配了登录页面逻辑，实现了令牌的 `localStorage` 持久化。
  - 根据后端规范适配了用户信息展示（`realName`）及权限控制（`access.ts`）。

- **认证与鉴权修复**：
  - **令牌注入**：修复了 JWT 令牌中缺失 `role_id` 的问题。现在登录成功后，令牌会包含用户的首选角色 ID。
  - **网关健壮性**：修复了 `AuthFilter` 中的空指针异常（NPE），增加了 Token 信息合法性校验。
  - **权限比对修正**：修正了网关层对权限字符（如 `sys:user:list`）与请求路径之间的比对逻辑，确保权限校验生效。

- **基础设施优化**：
  - **统一日志增强**：重定向了后端容器日志至 `logs/backend.log`，前端 Umi 日志至 `logs/frontend.log`。
  - **启动脚本升级**：更新了 `z-run.sh`，增强了对 Umi/Max 进程的动态清理能力，并优化了环境变量加载。
  - **环境健康检查**：修正了 Docker 容器的 `HealthCheck` 指标，解决了服务在 Docker 中显示 `unhealthy` 的误判问题。

## 待办事项
- 进一步优化 Ant Design Pro 的菜单动态加载逻辑（从数据库获取）。
- 完善 Token 过期的平滑刷新机制。
