# Release v1.0.7 (2026-04-16)

## 🚀 核心更新 (Core Updates)
本次更新实现了 **基础设施 (Infrastructure)** 模块的全栈落地，包括字典管理、参数配置、通知公告及文件中心，并引入了高性能的二级缓存机制。

### 1. 基础设施模块实现
- **字典管理**：重构为 `Type-Data` 双表模型，支持分类管理与键值维护，内置缓存预热逻辑。
- **参数配置**：支持全局业务参数的在线维护，集成 Redis 缓存，实现系统策略的秒级生效。
- **通知公告**：实现门户公告、系统通知的发布与生命周期管理，支持富文本排版。
- **文件中心**：构建文件元数据追溯体系，支持图片预览、大小格式化及多协议存储查询。

### 2. 技术与性能优化
- **Redis 二级缓存**：在 `zephyr-system-biz` 中深度集成 Redis，针对高频访问的字典项（/dict/data/type）与配置项（/config/key）实施缓存策略。
- **依赖治理**：统一使用 Spring 原生 `StringUtils` 替代外部依赖，精简打包体积并修复编译兼容性问题。

## 🛠️ 数据库变更 (Database Changes)
- 新增表：`sys_dict_type`, `sys_dict_data`, `sys_config`, `sys_file`, `sys_notice`。
- 初始化：同步注入系统基础设置与演示数据。

## 🔗 代码变动 (Code Changes)
- **后端**：新增 `com.zephyr.system` 下的 5 套 Entity、Mapper、Service、Controller 骨架及业务逻辑。
- **前端**：
  - 新增 `infra` 导航组及权限路由。
  - 实现 `dict`, `config`, `notice`, `file` 四大交互页面。
  - 新增 `infra.ts` Service 层定义。
