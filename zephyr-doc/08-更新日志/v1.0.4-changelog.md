# v1.0.4 更新日志

**发版时间**: 2026-04-16

## 后端变更 (zephyr-server)
- **资产迁移**: 清理了 `z-script/sql/` 目录下散乱的原始 SQL 脚本，统一移至 `zephyr-doc` 进行标准化版本管理。
- **权限安全**: 优化了 `JwtAuthenticationFilter` 的鉴权处理逻辑（具体变动详见 Git Diff）。

## 前端变更 (zephyr-web)
- **环境维护**: 更新了前端运行时的状态标记文件。

## 文档变更 (zephyr-doc) - [核心重构]
- **目录体系重组**: 全面推行 `01-08` 编号制目录结构，提升文档检索效率。
- **数据库管理体系**:
    - 确立“一模块一文件”扁平化管理原则。
    - 脚本分类按“基础设施”、“权限核心”、“审计”、“平台配置”技术维度重组。
    - 对安全审计模块（`sys_login_log`, `sys_oper_log`）执行合规性修复，补全审计字段并优化索引。
- **开发规范建立**:
    - 新建 `05-开发规范` 模块，明确了数据库建模的具体约束。
    - 建立了 **AI 强制预读机制**，要求 AI 在处理任务前必须先读取规范文件。
    - 制定了 `01-代码提交与发版规范`，实现版本发布流程的自动化与标准化。
- **功能设计优化**: 将功能设计细分为“系统功能”与“业务功能”，并建立了 00-总览引导机制。
