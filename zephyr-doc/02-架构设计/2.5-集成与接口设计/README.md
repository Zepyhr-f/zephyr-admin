# 2.5-集成与接口设计

本目录用于沉淀对外/对内接口与集成规范：REST API 约定、错误码、RPC 契约（Feign）、事件与消息（可选）。

---

## 文件清单

| 文件 | 关键词（便于 AI 检索） | 说明 |
|---|---|---|
| [01-API规范与错误码.md](./01-API规范与错误码.md) | REST、分页、错误码、R<T> | API 协议与错误码号段规范 |
| [02-RPC契约与Feign规范.md](./02-RPC契约与Feign规范.md) | Feign、契约、DTO、版本演进 | `zephyr-api/*` 使用边界与演进策略 |
| [03-事件与消息设计.md](./03-事件与消息设计.md) | MQ、事件、topic、幂等 | 消息/事件规范（未引入 MQ 时作为预留） |

## 阅读建议
- 对外接口协作优先看《API 规范与错误码》。
- 涉及服务间调用边界时，再看《RPC 契约与 Feign 规范》。
- 准备引入消息队列或异步事件时，查看《事件与消息设计》。

## 维护规则
- 本目录维护接口契约和集成边界，避免把具体业务接口清单散落到多个目录。
- 响应体、错误码、分页协议变化时，必须优先更新《API 规范与错误码》。
- 尚未落地的消息能力可以保留为预留设计，但需明确标注状态，避免误认为已上线。
