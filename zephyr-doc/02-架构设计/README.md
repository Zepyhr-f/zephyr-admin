## 维护规则
- 架构设计目录只维护长期稳定的结构、职责边界和关键取舍，不承载短期任务排期。
- 需要表达未来方向时，可在现有文档中使用“演进方向 / 待补充”区块，但不得创建不存在的目录入口。
- 鉴权链路、安全基线等跨端主题优先收敛到对应 SSOT 文档，避免在多个子目录重复定义。

## 修改指引
- 当服务发生重大架构调整（如新增微服务、引入消息队列）时，请更新 `2.1-后端设计/` 下的对应文档。
- 网关路由、Nacos 配置等具体技术实现可参见 `06-技术分享/` 目录中的专项说明。
