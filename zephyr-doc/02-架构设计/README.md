# 03-架构设计

本模块从技术层面描述系统的整体架构布局，按前后端维度分目录管理系统设计文档，确保架构演进过程有据可查。

## 📁 子目录说明

### [3.1-后端设计](./3.1-后端设计/)
记录后端服务工程的分层架构、核心模块职责、认证流程及技术选型。

| 文件 | 说明 |

### [3.2-前端设计](./3.2-前端设计/)
记录前端工程的工程结构、组件规范、状态管理及与后端服务的对接约定（待补充）。

## 修改指引
- 当服务发生重大架构调整（如新增微服务、引入消息队列）时，请更新 `3.1-后端设计/` 下的对应文档。
- 网关路由、Nacos 配置等具体技术实现可参见 `06-技术分享/` 目录中的专项说明。
