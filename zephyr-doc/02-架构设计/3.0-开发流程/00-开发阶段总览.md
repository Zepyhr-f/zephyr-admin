# Zephyr Admin 统一开发流程总览

> 本文档定义前后端并行协作的完整开发路径。每个阶段都有独立的**后端任务文件**和**前端任务文件**，两端在同一阶段内并行推进，通过**接口契约（OpenAPI）**作为唯一协作边界。

---

## 核心协作原则

| 原则 | 说明 |
|---|---|
| **契约先行** | Phase 2 先由后端定义 OpenAPI 契约，前端基于 MSW Mock 独立开发，彻底解除互相等待 |
| **阶段同步** | 每个阶段结束有明确的**验收标准**，两端对齐后才进入下一阶段 |
| **各司其职** | 后端只关心 API 正确性与性能；前端只关心 UI 体验与交互质量 |
| **统一规范** | 所有人遵守同一套 Commit 规范（Conventional Commits）和 Code Review 流程 |

---

## 开发阶段总览

```
Phase 0  ─── 环境搭建
              后端：JDK 21 + Maven + Docker/Nacos 本地服务
              前端：Node 20 + pnpm + VSCode + MSW Mock

Phase 1  ─── 工程脚手架
              后端：Maven BOM + 模块骨架 + R<T> 统一响应体
              前端：src/ 目录结构 + 设计系统 Token + 主题配置

Phase 2  ─── 接口契约（关键协作节点）
              后端：设计 OpenAPI 规范 + Smart-Doc + 数据库迁移脚本
              前端：Axios 封装 + TanStack Query 规范 + MSW Handler 编写

Phase 3  ─── 认证与权限体系
              后端：双令牌发行 + Spring Security + RBAC 权限缓存
              前端：登录页 + Token 管理 + 路由守卫 + 动态菜单

Phase 4  ─── 核心业务模块
              后端：System 模块 CRUD API（用户/角色/菜单/字典/部门）
              前端：系统管理页面（列表 + 表单 + 详情）

Phase 5  ─── 测试、质量与上线
              后端：单元测试 + 集成测试 + CI/CD 流水线
              前端：组件测试 + E2E + 构建优化 + 部署配置
```

## 阶段验收清单（两端对齐节点）

| 阶段结束 | 验收标准 |
|---|---|
| Phase 0 完成 | 两端本地均可启动服务，接口健康检查通过 |
| Phase 1 完成 | 两端目录结构搭建完成，基础工具链运行正常 |
| Phase 2 完成 | OpenAPI 文档已发布，前端 MSW Mock 覆盖所有核心接口 |
| Phase 3 完成 | 登录→鉴权→权限缓存全链路联调通过 |
| Phase 4 完成 | System 模块所有 CRUD 接口 + 页面前后端联调通过 |
| Phase 5 完成 | 测试覆盖率达标，CI/CD 流水线跑通，生产部署成功 |

## Git 工作流规范（全员统一）

```
main            # 生产分支，只接受 merge request，不直接 push
  └─ dev        # 集成分支，阶段完成后合并至 main
       └─ feature/xxx  # 功能分支（前后端各自开），命名格式：feature/<scope>/<desc>
       └─ fix/xxx      # Bug 修复分支
```

Commit 格式（Conventional Commits）：
```
feat(auth): 实现双令牌刷新接口
fix(user): 修复角色分配后权限缓存未刷新的问题
docs(api): 更新用户管理接口的 Smart-Doc 注释
refactor(gateway): 将 AuthFilter 中 synchronized 改为 ReentrantLock
```
