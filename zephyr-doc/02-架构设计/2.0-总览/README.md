# 2.0-总览

本目录用于回答“系统是什么样”的问题：系统边界、核心组件、关键链路、以及关键架构决策（ADR）。  
建议所有新同学从这里开始阅读。

---

## 文件清单

| 文件 | 关键词（便于 AI 检索） | 说明 |
|---|---|---|
| [00-系统架构总览.md](./00-系统架构总览.md) | C4、组件图、部署形态、核心链路 | 系统全景与关键链路（总入口） |
| [01-术语表与边界.md](./01-术语表与边界.md) | 术语、边界、scope、in/out | 统一口径与系统边界（避免歧义） |
| [02-关键设计决策ADR.md](./02-关键设计决策ADR.md) | ADR、设计决策、为什么 | 记录关键决策与取舍，防止口径漂移 |

## 阅读建议
- 新同学优先从《系统架构总览》开始建立全景认知。
- 对概念边界有歧义时，再看《术语表与边界》统一口径。
- 想知道“为什么这样设计”时，查看《关键设计决策 ADR》。

## 维护规则
- 本目录是架构层总入口，优先维护长期稳定的边界、关系和关键取舍。
- 系统整体形态发生变化时，应优先更新《系统架构总览》。
- 术语变化、边界调整、重大架构取舍应同步更新本目录对应文档。
