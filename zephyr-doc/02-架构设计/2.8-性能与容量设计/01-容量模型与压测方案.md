## 1. 统一口径
- 并发（Concurrency）：同一时刻正在处理的请求数
- QPS：每秒请求数
- RT：响应时间（建议关注 P95/P99）

近似关系（Little’s Law）：
> 并发 ≈ QPS × 平均 RT（秒）

---

## 2. 压测对象（建议分层）
1. 网关层压测：路由 + JWT 校验 + URL-RBAC
2. 平台服务压测：登录/权限查询/系统管理 CRUD
3. 业务服务压测：订单/商品核心接口
4. 全链路压测：网关 → 平台/业务 → DB/Redis

---

## 3. 指标与验收（示例）
- P95 < 200ms（按接口分级）
- 错误率 < 0.1%
- 压测期间无连接池耗尽、无明显 GC 抖动

---

## 4. 工具建议
JMeter / Gatling / k6（择一）；指标采集结合 Prometheus/Grafana。
