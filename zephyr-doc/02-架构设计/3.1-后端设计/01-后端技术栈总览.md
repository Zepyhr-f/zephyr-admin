# 后端技术栈总览

> 本文是 Zephyr Admin 后端的**顶层选型地图**，说明每项技术的选型理由及版本要求。所有子模块必须遵守此清单，不得自行引入未经评审的依赖。

---

## 1. 核心语言与框架

| 类别 | 技术 | 版本 | 选型理由 |
|---|---|---|---|
| 语言 | **Java** | 21 LTS | 虚拟线程（Project Loom）大幅提升 I/O 并发，`Record`/`Pattern Matching` 简化代码 |
| 基础框架 | **Spring Boot** | 3.4.x | 自动配置、原生 GraalVM 支持、Actuator 开箱即用 |
| 微服务套件 | **Spring Cloud** | 2023.0.x | 官方维护，与 Nacos 完美集成，提供统一服务治理 |
| 构建工具 | **Maven** | 3.9+ | 多模块 BOM 管理，`--fail-at-end` 支持并行构建 |

---

## 2. 基础设施与中间件

| 组件 | 版本 | 作用 |
|---|---|---|
| **Spring Cloud Gateway** | — | 唯一对外入口：路由转发、JWT 鉴权过滤、限流熔断 |
| **Nacos** | 2.x | 服务注册发现 + 动态配置热更新 |
| **MySQL** | 8.0 / OceanBase | 主关系型数据库，InnoDB 事务 + JSON 列支持 |
| **Redis** | 7.2 | Token 存储、角色权限热缓存、语义缓存、分布式锁 |
| **MinIO** | 最新稳定版 | 本地 S3 兼容对象存储，生产可切换阿里云 OSS |
| **Docker + OrbStack** | — | 本地容器化依赖（macOS 推荐 OrbStack，替代 Docker Desktop） |

---

## 3. 持久化与 ORM

| 技术 | 版本 | 说明 |
|---|---|---|
| **MyBatis-Plus** | 3.x | 自动填充审计字段、逻辑删除、分页插件、乐观锁 `@Version` |
| **MapStruct** | 1.6.x | 编译期生成 DTO ↔ Entity 映射代码，**零反射**，替代 BeanUtils |
| **Flyway** | 10.x | SQL 脚本式数据库版本迁移，CI 中自动执行 |
| **Hibernate Validator** | 8.x | Bean Validation 参数校验，统一错误消息 |

> **为什么用 MapStruct 而非 BeanUtils？**
> MapStruct 在编译期生成 Java 代码，字段不匹配直接编译报错，运行期零反射损耗；BeanUtils 靠反射，字段拼写错误运行期才发现。

---

## 4. 安全与鉴权

| 技术 | 说明 |
|---|---|
| **Spring Security 6** | 过滤器链管理，无状态会话配置 |
| **JWT（双令牌机制）** | Access Token（30 min，Header 传输）+ Refresh Token（7 天，HttpOnly Cookie） |
| **ES256 / HS256** | JWT 签名算法可配置，推荐 ES256（非对称，私钥签名） |
| **BCrypt** | 密码存储，自带盐值，防止彩虹表攻击 |
| **Redis 黑名单** | 将失效 token 的 `jti` 写入 Redis，实现**秒级强制下线** |

---

## 5. API 文档

| 技术 | 说明 |
|---|---|
| **Smart-Doc** | 基于标准 JavaDoc 注释生成文档，**零注解污染**，替代 Swagger/Knife4j |

> **为什么不用 Swagger？**
> Swagger 需要大量 `@Schema`、`@ApiModel` 注解侵入业务代码；Smart-Doc 只读 JavaDoc，代码更纯粹，CI 中自动生成 HTML/Markdown 文档。

## 9. 技术选型决策说明

| 决策 | 原因 |
|---|---|
| **Java 21 虚拟线程** | I/O 密集型场景（DB 查询、HTTP 调用）吞吐提升数倍，无需改变编程模型 |
| **双令牌替代单 Token** | Access Token 短寿命降低泄漏风险；Refresh Token HttpOnly Cookie 防 XSS；Redis 黑名单实现强制下线 |
| **MapStruct 替代 BeanUtils** | 编译期类型安全，运行期零损耗 |
| **Smart-Doc 替代 Swagger** | 代码零侵入，文档与注释同源，CI 自动生成 |
| **Flyway 统一迁移** | 脚本版本化，多人协作不再手动执行 SQL，CI 自动执行 |
| **Maven BOM** | 在 `zephyr-bom` 中统一锁定所有三方依赖版本，根除 Dependency Hell |
