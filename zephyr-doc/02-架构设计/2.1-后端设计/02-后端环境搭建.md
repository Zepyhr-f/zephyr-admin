> **目标**：让每位后端开发者在本机以完全一致、可复现的方式启动 Zephyr Admin 后端服务，从拉代码到接口可用不超过 15 分钟。

## 2. 版本锁定策略

> **原则**：JDK 版本不统一是后端最常见的"幽灵 Bug"来源，必须强制锁定。

### 2.1 Java 版本锁定

项目根目录提供 `.sdkmanrc` 文件：
```
java=21.0.2-open
```
所有人进入项目目录后执行 `sdk env install` 即可自动切换正确 JDK。

`pom.xml` 中同步声明编译目标版本：
```xml
<properties>
    <java.version>21</java.version>
    <maven.compiler.release>21</maven.compiler.release>
</properties>
```

### 2.2 Maven 依赖版本锁定（BOM 方案）

项目采用 `zephyr-bom` 子模块统一锁定所有三方依赖版本，彻底消除微服务多模块环境中的依赖冲突（Dependency Hell）。

```xml
<!-- 根 pom.xml 中引入 BOM -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>${spring-boot.version}</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-dependencies</artifactId>
            <version>${spring-cloud.version}</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
        <!-- MyBatis-Plus、MapStruct 等统一在此锁定 -->
    </dependencies>
</dependencyManagement>
```

### 2.3 Docker 镜像版本锁定

`docker-compose.yml` 中使用**固定 tag**，禁止使用 `latest`：
```yaml
services:
  mysql:
    image: mysql:8.0.38      # ✅ 固定版本
  redis:
    image: redis:7.2.4       # ✅ 固定版本
  nacos:
    image: nacos/nacos-server:v2.3.2
```

## 3. 本地依赖服务启动（Docker Compose）

### 3.1 需要的服务清单

| 服务 | 端口 | 说明 |
|---|---|---|
| **MySQL 8.0** | `3306` | 主数据库 |
| **Redis 7.2** | `6379` | 缓存与 Token 存储 |
| **Nacos 2.x** | `8848` / `9848` | 注册中心与配置中心 |
| **MinIO** | `9000` / `9001` | 本地对象存储（文件上传） |

### 3.2 启动方式

```bash
# 启动所有后端依赖服务
./z-server.sh infra:up

# 查看运行状态
./z-server.sh infra:status

# 停止所有服务
./z-server.sh infra:down
```

## 4. 项目初始化步骤

```bash
# 1. 克隆仓库
git clone https://github.com/your-org/zephyr-admin.git
cd zephyr-admin/zephyr-server

# 2. 切换正确的 JDK
sdk env install

# 3. 安装依赖并构建（跳过测试加快速度）
mvn clean install -DskipTests

# 4. 启动数据库迁移（Flyway 自动执行）
mvn flyway:migrate -Dspring.profiles.active=dev

# 5. 启动服务（dev 环境）
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

启动成功后访问 `http://localhost:9000/actuator/health`，应返回 `{"status":"UP"}`。

## 5. 虚拟线程开启

Java 21 虚拟线程在 Spring Boot 3.2+ 中只需一行配置即可开启：

```yaml
# application.yml
spring:
  threads:
    virtual:
      enabled: true
```

**注意事项**：
- HikariCP 需调大 `maximum-pool-size`（建议从默认 10 调至 50~200，根据并发量压测确定）
- 所有 `synchronized` 块须改为 `ReentrantLock`，否则会出现**线程固定**（Pinning）问题导致性能下降

## 7. Git Hooks 规范

项目使用 **Lefthook** 管理 Git Hooks，`mvn install` 后自动安装。

| Hook | 触发时机 | 执行内容 |
|---|---|---|
| `pre-commit` | `git commit` 前 | Checkstyle / SpotBugs 扫描，失败则阻止提交 |
| `commit-msg` | 输入提交信息后 | Commitlint 格式检查（Conventional Commits） |

Commit 格式示例：
```
feat(auth): 新增双令牌刷新机制
fix(system): 修复用户角色分配后权限缓存未刷新的问题
refactor(gateway): 将 AuthFilter 中 synchronized 替换为 ReentrantLock
```

## 8. 验收标准

环境搭建完成后，满足以下所有条件才算完成：

- [ ] `mvn clean install -DskipTests` 无报错
- [ ] `http://localhost:9000/actuator/health` 返回 `{"status":"UP"}`
- [ ] `POST /auth/login` 可正常返回 Access Token 与 Refresh Token Cookie
- [ ] IntelliJ IDEA 中无红色报错，Lombok/MapStruct 注解正常识别
- [ ] `git commit` 时 Lefthook 正常触发
