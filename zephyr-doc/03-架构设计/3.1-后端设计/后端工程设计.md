# 后端工程设计

本文档基于对 `zephyr-server` 源代码的实际分析，系统梳理了项目的后端工程结构、核心模块职责、关键技术机制以及服务间的协作方式。

> **关联代码目录**：[`zephyr-server/`](../../../zephyr-server/)

---

## 1. 工程模块分层总览

`zephyr-server` 采用 **Spring Cloud 微服务架构**，按"基础设施层 → 公共服务层 → 业务层 → 网关层"进行分层组织。

```
zephyr-server/
├── zephyr-gateway/          # 统一网关层：所有请求的唯一入口
├── zephyr-common/           # 公共服务层：鉴权、系统管理、运维、代码生成
│   ├── common-boot/         # 公共服务聚合启动器（统一启动 Auth + System）
│   ├── zephyr-auth/         # 认证授权服务（登录/登出/用户信息）
│   ├── zephyr-system/       # 系统管理服务（RBAC 核心：用户/角色/菜单/部门/字典）
│   ├── zephyr-ops/          # 系统监控服务（CPU/内存监控）
│   └── zephyr-support/      # 开发支撑服务（代码生成器）
├── zephyr-base/             # 基础设施层：通用能力封装（SDK/Starter）
│   ├── zephyr-core/         # 核心工具：R 响应体、工具类、基础控制器
│   └── zephyr-starter/      # 能力 Starter：JWT、MyBatis-Plus、Redis、Auth
└── zephyr-biz/              # 业务服务层（预留扩展）
```

---

## 2. 基础设施层（zephyr-base）详解

本层是所有业务服务依赖的基础 SDK，提供标准化能力封装，**不含任何业务逻辑**。

### 2.1 zephyr-core（核心工具）

| 模块               | 关键类/文件        | 功能描述                                                                    |
| :----------------- | :----------------- | :-------------------------------------------------------------------------- |
| `zephyr-core-tool` | `R<T>`             | **统一响应体**，封装 `code`, `success`, `data`, `msg`，所有接口均返回此对象 |
| `zephyr-core-tool` | `ResultCode`       | 全局状态码枚举（200=成功，等）                                              |
| `zephyr-core-tool` | `WebConstants`     | Web 层通用常量（请求头键名等）                                              |
| `zephyr-core-boot` | `ZephyrController` | 业务控制器基类，提供获取当前用户会话方法                                    |
| `zephyr-core-boot` | `UserSession`      | 封装当前请求的用户上下文（从 Header 中解析）                                |

### 2.2 zephyr-starter（能力 Starter）

| Starter 模块           | 关键组件                     | 功能描述                                                                       |
| :--------------------- | :--------------------------- | :----------------------------------------------------------------------------- |
| `zephyr-starter-jwt`   | `JwtUtil` / `JwtKeyProvider` | JWT 创建与解析，支持 HS256 与 ES256 双算法切换，通过 `JwtProviderFactory` 选择 |
| `zephyr-starter-jwt`   | `JwtConfig` / `JwtConstant`  | JWT 配置（过期时间、Header 键名 `Authorization`、前缀 `Bearer `）              |
| `zephyr-starter-mp`    | `BaseEntity`                 | **所有业务表的基类**，含 id、五大审计字段，通过 `@TableLogic` 自动逻辑删除     |
| `zephyr-starter-mp`    | `MyBatisPlusHandler`         | 自动填充 `create_user`、`update_user`、`create_time`、`update_time`            |
| `zephyr-starter-mp`    | `PageQuery`                  | 统一分页查询参数封装                                                           |
| `zephyr-starter-redis` | `RedisUtil`                  | Redis 操作工具类（String/Set/过期等常用操作）                                  |
| `zephyr-starter-redis` | `RedisConstant`              | Redis Key 常量（`TOKEN_PREFIX`、`ROLE_PREFIX` 等）                             |

## 3. 网关层（zephyr-gateway）详解

网关是所有外部请求的唯一入口，基于 **Spring Cloud Gateway** 实现，提供请求路由与身份认证。

### 3.1 认证过滤器（AuthFilter）设计

`AuthFilter` 实现了 `GlobalFilter`，过滤器优先级最高（`order = -1`）。

**处理流程：**

```
请求进入
  └─ 是否在白名单？ (isSkip：DefaultSkipProp + authProperties.whiteApi)
       ├─ 是 → 直接放行
       └─ 否 → 读取 Authorization Header
                 └─ Header 存在？
                      ├─ 否 → 返回 401 Token缺失
                      └─ 是 → 解析 JWT + 比对 Redis 存储的真实 Token
                               └─ Token 有效且未过期？
                                    ├─ 否 → 返回 401 Token无效或已过期
                                    └─ 是 → 提取 role_id 确认角色信息存在
                                              └─ role_id 存在？ → 放行（细粒度鉴权交由后端服务自行控制）
```

> **设计意图**：网关**只做 Token 有效性验证**，不做路径级别的权限比对（因为 `perms` 的值是语义标识如 `sys:user:list`，与 URL 路径格式不同，在网关层比对会误拦截）。

### 3.2 白名单配置

白名单来源于两处：

1. **硬编码白名单**（`DefaultSkipProp`）：`/login`、`/logout`、健康检查等。
2. **动态白名单**（Nacos 配置 `authProperties.whiteApi`）：可热更新，用于灵活配置。

## 4. 认证授权服务（zephyr-auth）详解

### 4.1 API 端点列表

| 方法   | 路径      | 功能                                          | 是否需要 Token |
| :----- | :-------- | :-------------------------------------------- | :------------- |
| `POST` | `/login`  | 用户名密码登录，返回 JWT Token                | ❌ 白名单      |
| `GET`  | `/info`   | 从 Token 提取用户身份，查询完整角色与权限清单 | ✅ 需要        |
| `GET`  | `/logout` | 删除 Redis 中的 Token 使其失效                | ✅ 需要        |

### 4.2 登录核心流程

```
POST /login { username, password }
  ├─ Spring Security AuthenticationManager.authenticate()
  │    └─ ZephyrUserDetailsService.loadUserByUsername()
  │         └─ Feign 调用 zephyr-system 服务 → 查 sys_user + sys_user_role
  │
  ├─ 认证通过 → 构建 Claims：{ userId, username, role_id(取第一个主要角色) }
  ├─ JwtUtil.generateToken(claims) → 生成 JWT（含签名算法 + 过期时间）
  ├─ RedisUtil.setString("token:{username}", token, 1小时) → 持久化到 Redis
  └─ 返回 LoginResponse { token, refreshToken(暂同 token), userId }
```

### 4.3 Token 格式与 Redis Key 规则

| 项目             | 说明                            |
| :--------------- | :------------------------------ |
| Token Header     | `Authorization: Bearer <token>` |
| JWT Claims       | `userId`, `username`, `role_id` |
| Redis Key Format | `zephyr:token:{username}`       |
| Token 有效期     | 1 小时（登录时写入 Redis）      |

## 5. 系统管理服务（zephyr-system）详解

### 5.1 对外 API 模块

| 控制器           | 路由前缀 | 核心功能                                   |
| :--------------- | :------- | :----------------------------------------- |
| `UserController` | `/user`  | 用户增删改查、状态更新、重置密码、分配角色 |
| `RoleController` | `/role`  | 角色管理、权限分配                         |
| `MenuController` | `/menu`  | 菜单树维护、权限标识管理                   |
| `DeptController` | `/dept`  | 部门树管理                                 |
| `DictController` | `/dict`  | 数据字典维护                               |

### 5.2 RBAC 权限缓存机制

`RolePermsLoader` 实现了 `ApplicationRunner`，在服务**启动时自动预热**角色权限缓存：

```
服务启动
  └─ 遍历所有角色（sys_role）
       └─ 对每个角色：查 sys_role_menu JOIN sys_menu.perms → 获取权限标识集合
            └─ Redis SET：Key="role:{roleId}", Value=Set<perms标识>
```

**设计目的**：将热数据预加载到 Redis，接口鉴权时无需每次查库，直接从 Redis 读取当前用户角色对应的权限清单进行比对。

### 5.3 服务间调用（Feign）

`zephyr-auth` 服务通过 Feign Client 调用 `zephyr-system` 服务：

| 调用方        | Feign Client  | 调用目标                            | 说明                                       |
| :------------ | :------------ | :---------------------------------- | :----------------------------------------- |
| `zephyr-auth` | `IUserClient` | `UserController` in `zephyr-system` | 登录时查询用户、获取角色列表、获取权限标识 |

## 6. 关键技术选型汇总

| 技术栈     | 组件                     | 用途                       |
| :--------- | :----------------------- | :------------------------- |
| 微服务框架 | Spring Cloud             | 服务注册/发现/配置         |
| 网关       | Spring Cloud Gateway     | 统一入口、身份认证         |
| 配置中心   | Nacos                    | 动态路由与白名单配置       |
| 鉴权       | Spring Security + JWT    | 无状态身份认证             |
| 数据库 ORM | MyBatis-Plus             | 自动填充、逻辑删除、分页   |
| 缓存       | Redis                    | Token 存储、角色权限热缓存 |
| API 文档   | Knife4j (基于 Swagger 3) | 在线接口预览与调试         |
| Token 算法 | HS256 / ES256（可配置）  | JWT 签名与验证             |

## 7. 已知设计备注与待优化项

| 项目             | 现状描述                                                         | 建议后续方向                                                             |
| :--------------- | :--------------------------------------------------------------- | :----------------------------------------------------------------------- |
| **RefreshToken** | 目前 `refreshToken` 与 `token` 使用相同值                        | 独立实现长效 RefreshToken 机制，支持无感刷新                             |
| **用户信息透传** | 网关中已注释掉向下游透传 userId/username 的 Header 代码          | 按需开启，配合 `UserSession` 简化业务代码的用户获取逻辑                  |
| **细粒度鉴权**   | 当前细粒度权限控制的具体实现位置待确认（网关不做，后端服务自控） | 在 `ZephyrController` 层引入 `@PreAuthorize` 注解配合 Redis 权限缓存完成 |
| **多角色主角色** | 登录 Token 中的 `role_id` 仅取用户角色列表第一个                 | 改为携带完整 `role_ids` 列表，或设计角色优先级字段                       |
