# 3.2 前端设计 (Frontend Design)

## 一、 技术栈选型
基于 **Ant Design Pro 6** (Umi 4) 体系构建，采用 React 18 与 TypeScript 进行开发。

-   **核心框架**: [UmiJS 4.x](https://umijs.org/) (插件化企业级前端应用框架)
-   **组件库**: [Ant Design 5.x](https://ant.design/) (企业级 UI 设计语言)
-   **高级组件**: [ProComponents](https://procomponents.ant.design/) (重度抽象的业务组件库)
-   **数据流**: [@umijs/max (Valtio/Model)](https://umijs.org/docs/max/data-flow)
-   **网络请求**: [Axios](https://github.com/axios/axios) (集成在 @umijs/max request 插件中)
-   **样式方案**: Less + CSS-in-JS (Antd 5 默认)

---

## 二、 目录结构规范
项目遵循 Umi 4 的典型目录结构，并按业务模块进行垂直拆分：

```text
src/
├── access.ts            # 权限定义文件 (对应 initialState.canAdmin 等)
├── app.tsx               # 运行时配置 (包含 getInitialState, layout 配置)
├── components/           # 全局公共组件
│   ├── Footer/           # 页脚组件
│   └── RightContent/     # 顶部导航右侧区域 (头像、多语言、搜索)
├── locales/              # 国际化资源 (zh-CN, en-US 等)
├── pages/                # 页面目录 (按功能模块划分)
│   ├── system/           # 系统管理 (用户、角色、菜单、部门)
│   ├── audit/            # 安全审计 (日志记录、在线用户)
│   ├── monitor/          # 系统监控 (服务状态、缓存、任务)
│   ├── tool/             # 开发工具 (代码生成、接口预览)
│   └── user/             # 认证相关 (登录、注册)
├── services/             # 后端接口定义 (API 层)
│   ├── system/           # 与业务模块对应的后端接口
│   └── ant-design-pro/   # 基础设施接口 (如获取当前用户信息)
├── global.less           # 全局样式
└── requestErrorConfig.ts # 统一请求拦截与异常处理配置
```

## 三、 核心模块功能设计

### 1. 认证模块 (Authentication)
-   **位置**: `src/pages/user/`
-   **功能**: 登录验证、Token 存储、自动重定向。
-   **逻辑**: 登录成功后，Token 存储在 Cookie/LocalStorage 中，并通过 `getInitialState` 初始化用户信息。

### 2. 系统管理 (Core Admin)
-   **用户管理** (`pages/system/user`): 基于 `ProTable` 实现，支持复杂的筛选、导出、CRUD。
-   **权限体系**: 基于 `access.ts` 配合后台返回的 `perms` 权限标识，实现按钮级别的隐藏/禁用。

### 3. 开发工具 (Developer Tools)
-   **代码生成** (`pages/tool/gen`): 
    -   主页面展示表清单。
    -   `edit` 子页面进行详细配置。
    -   `PreviewModal` 实现生成的 Java/TS/SQL 代码实时预览与一键复制。

## 四、 关键技术细节

### 1. 自动路由生成
项目在 `config/routes.ts` 中集中管理路由，采用侧边栏侧边导航 (`layout: 'mix'`)。所有路由权限通过 `access` 字段进行校验。

### 2. 请求拦截器
在 `src/requestErrorConfig.ts` 中定义：
-   **请求前缀**: 自动识别环境，开发环境代理至网关 (8001)。
-   **鉴权头**: 自动在 Header 中注入 `Authorization: Bearer <Token>`。
-   **错误码处理**: 统一处理 401 (过期自动登出)、403 (无权访问) 等业务状态码。

### 3. 表格开发标准
项目统一使用 `@ant-design/pro-table`：
-   自带分页查询、列隐藏、密度调整功能。
-   列配置采用 JSON 驱动模式，减少样板代码。
-   操作列统一收纳至下划线菜单或 `TableDropdown`。

---

## 五、 设计约束
1. **禁止直接操作 DOM**: 必须遵循 React 声明式编程。
2. **样式隔离**: 业务页面优先使用 `xxx.style.ts` 或 CSS Modules。
3. **严禁硬编码**: 所有接口路径、字典状态必须统一管理（Services/Locales）。
