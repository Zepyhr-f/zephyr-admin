## 0. 前置条件（开始前请确认）

> [!IMPORTANT]
> 以下版本口径统一以《00-项目简介/00-版本与环境矩阵.md》为准；如你本地环境与其不一致，请先对齐版本再排查问题。

### 0.1 必备工具
- 后端：Go 1.23+、Docker（或 OrbStack）。
- 前端：Node、pnpm。
- 通用：Git、可用的终端环境。

### 0.2 常见问题（权限/可执行）
如果 `z-server.sh` 无法执行，请先授予执行权限：
```bash
chmod +x z-server.sh z-web.sh
```

## 1. 快捷联调方式 (推荐)

在项目根目录下，我们提供了统一的启停脚本 `z-server.sh` 和 `z-web.sh`。

### 1.1 一键启动后端
```bash
./z-server.sh start
```
该命令会自动在后台启动所有后端微服务（Gateway, Auth, Identity 等）。

### 1.2 一键停止后端
```bash
./z-server.sh stop
```

### 1.3 一键重启后端
```bash
./z-server.sh restart
```

## 2. 独立模块调试

### 2.1 后端调试 (Go)
- 环境要求：Go 1.23+、Docker（版本以《版本与环境矩阵》为准；或各组件独立安装）。
- 调试建议：使用 GoLand 或 VSCode 分别进入 `zephyr-go/app/...` 目录下执行 `go run` 启动微服务。

### 2.2 前端调试 (React)
- 进入目录：`cd zephyr-web`
- 安装依赖：`pnpm install`
- 启动服务：`pnpm dev`
- 浏览器访问：以终端输出为准（Vite 默认 `http://localhost:5173`，如项目改为 `8000` 请以配置为准）

---

## 3. 启动后验证（建议按顺序）

1. **前端页面可访问**：浏览器打开 Vite 地址，能看到登录页或首页。
2. **后端服务健康**：确认 Gateway/Auth/Identity 等服务已启动且无报错日志。
3. **联调验证**：
   - 关闭前端 Mock（如有），发起一次真实登录或拉取菜单/用户信息的请求；
   - 验证鉴权失败时能返回统一错误结构（例如 401/403）。

> [!TIP]
> 如果你在做纯前端开发，可以优先开启 MSW Mock，减少对后端依赖（详见《02-架构设计/2.2-前端设计/02-前端环境搭建.md》）。

## 5. 常见问题排障（Troubleshooting）

### 5.1 端口冲突
- 现象：启动时提示端口被占用。
- 处理：修改本地端口占用或调整配置；端口约定请登记到《版本与环境矩阵》。

### 5.2 Docker / 基础服务未就绪
- 现象：后端启动报错（无法连接 MySQL/Redis/Nacos）。
- 处理：先确认基础服务容器已正常启动，再启动后端服务。

### 5.3 前端 401/403 或跨域
- 现象：请求被拒绝或跨域失败。
- 处理：确认 Token 是否注入、后端鉴权白名单配置是否正确；跨域建议统一在网关层处理。
