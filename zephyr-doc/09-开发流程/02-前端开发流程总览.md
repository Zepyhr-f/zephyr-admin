# 前端开发流程总览

> 本文是前端开发的"地图"，定义了从零到上线的全部阶段划分、各阶段目标与产出物。后续每个阶段都有独立的详细文档与之对应。

---

## 1. 开发哲学

在动手写任何代码之前，先明确三条核心原则：

| 原则 | 说明 |
|---|---|
| **结构优先** | 先设计目录结构与分层，再实现功能，避免后期重构成本 |
| **设计驱动** | 先定义设计系统（Design Token），再写 UI，保证视觉一致性 |
| **契约先行** | 前后端先对齐接口契约（OpenAPI），再并行开发，减少联调阻力 |

## 3. 各阶段详细说明

### Phase 0 — 环境搭建
**目标**：让所有开发者在任何机器上能以完全一致的方式运行项目。

**核心任务**：
- 确认 Node.js / pnpm / Java 版本统一（`.nvmrc` / `engines` 字段锁定）
- 通过 OrbStack + Docker Compose 一键启动后端依赖（MySQL、Redis、Nacos）
- 配置 `.env.local` 本地环境变量（后端接口地址、Mock 开关等）
- 验证 `pnpm dev` 可正常启动，Mock 接口可正常响应

**产出物**：可运行的本地开发环境，附快速开始文档

---

### Phase 1 — 工程结构设计
**目标**：定义清晰的目录分层，让每个文件都有明确的"家"。

**核心任务**：
- 设计 `src/` 下的目录分层（pages / components / layouts / hooks / store / api / utils / types）
- 确定模块边界：哪些是全局的，哪些是页面私有的
- 制定文件命名规范（kebab-case vs PascalCase）
- 确定组件分层：UI 原子组件、业务组件、页面组件三层

**产出物**：工程结构设计文档 `04-前端工程结构设计.md`

---

### Phase 2 — 设计系统建立
**目标**：建立统一的视觉语言，让所有页面风格保持一致。

**核心任务**：
- 定义颜色系统（主色、中性色、语义色、品牌渐变）
- 定义排版规范（字体家族、字重、字号阶梯、行高）
- 定义间距系统（spacing scale 4px 基准）
- 定义 Ant Design Design Token 覆盖方案
- 定义 TailwindCSS v4 自定义 CSS 变量层

**产出物**：设计系统文档 `05-前端设计系统规范.md`

---

### Phase 3 — 路由架构搭建
**目标**：设计完整的路由结构，支持权限控制与懒加载。

**核心任务**：
- 定义路由层级（公开路由 vs 鉴权路由 vs 404/403 兜底路由）
- 设计动态路由方案（基于后端返回的菜单权限动态注册路由）
- 设计 Layout 组件结构（多套 Layout：管理后台 Layout / 全屏 Layout / 空白 Layout）
- 定义路由 Meta 信息规范（页面标题、面包屑、权限标识、是否缓存）
- 设计路由守卫逻辑（登录检查、权限检查、Token 刷新）

**产出物**：路由架构设计文档 `06-前端路由架构设计.md`

---

### Phase 4 — 状态管理设计
**目标**：明确哪些状态放在哪里，建立清晰的数据流。

**核心任务**：
- 区分服务端状态（TanStack Query 管理）与客户端状态（Zustand 管理）
- 设计全局 Store 模块划分（用户信息、权限、主题、多标签页）
- 定义 Store 命名与文件组织规范
- 设计数据持久化策略（哪些 State 需要 localStorage 持久化）

**产出物**：状态管理设计文档 `07-前端状态管理设计.md`

---

### Phase 5 — 接口层封装
**目标**：建立标准化的前后端通信层，与业务代码完全解耦。

**核心任务**：
- 设计 Axios 实例封装（统一 baseURL、请求头、超时）
- 设计请求/响应拦截器（Token 注入、统一错误弹窗、401 自动跳转登录）
- 定义接口目录组织规范（按业务模块拆分 API 文件）
- 设计 TanStack Query 的 QueryKey 命名规范
- 设计 Mock 数据方案（MSW Handler 组织结构）

**产出物**：接口层设计文档 `08-前端接口层设计.md`

---

### Phase 6 — 页面功能开发
**目标**：在完善的基础设施上，快速、规范地构建每个页面。

**核心任务**：
- 优先完成认证流程（登录页 → Token 存储 → 路由守卫 → 主页）
- 实现系统基础模块（用户管理、角色管理、菜单管理、字典管理）
- 按功能模块迭代开发，每个模块遵循：列表页 → 表单弹窗 → 详情页的开发顺序

**产出物**：各功能模块页面

---

### Phase 7 — 测试与质量保障
**目标**：保证代码质量，防止功能回归。

**核心任务**：
- 关键组件单元测试（Vitest + Testing Library）
- E2E 关键流程测试（Playwright：登录流程、权限验证）
- 性能优化：Bundle 分析（rollup-plugin-visualizer）、路由懒加载、组件 Memo

**产出物**：测试覆盖报告

---

### Phase 8 — 构建与部署
**目标**：将代码安全可靠地发布到生产环境。

**核心任务**：
- 配置生产构建优化（代码分割、Tree Shaking、静态资源压缩）
- 配置 Nginx 静态文件服务与 API 反向代理
- 配置 GitHub Actions CI/CD 流水线

**产出物**：生产可用的部署配置
