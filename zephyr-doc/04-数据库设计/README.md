# 04-数据库设计

本目录负责管理 Zephyr 项目的全部数据库资产。为了确保架构的严谨性与部署的顺滑性，将相关脚本划分为“建表”与“初始化”两个核心子模块。

## 📁 子模块概览

1. **[4.1-建表脚本 (DDL)](./4.1-建表脚本/)**: 定义系统的物理结构，包含所有业务表的创建语句。
2. **[4.2-初始化脚本 (DML/DCL)](./4.2-初始化脚本/)**: 包含项目运行所需的库环境配置、基础权限点及业务默认参数。

---

## 🏗️ 4.1-建表脚本模块详解

本模块采用“一子系统一文件”的设计策略。以下是各 SQL 文件的功能介绍及包含的表清单：

### 1. `4.1.1-rbac_core.sql` (权限体系核心)
**功能介绍**:
该文件定义了基于角色的访问控制 (RBAC) 模型。支持用户、角色、菜单、部门的关联，并提供了角色继承与互斥等高级安全特性。

**表清单**:
| 中文名称 | 英文名称 (Table Name) | 职责描述 |
| :--- | :--- | :--- |
| **部门表** | `sys_dept` | 维护企业组织架构树。 |
| **用户表** | `sys_user` | 存储账户核心信息（账号、密码、部门等）。 |
| **角色表** | `sys_role` | 定义角色标识及数据权限范围。 |
| **菜单权限表** | `sys_menu` | 维护页面路由与 API 权限标识。 |
| **用户角色关联表**| `sys_user_role` | 用户与角色的多对多映射。 |
| **角色菜单关联表**| `sys_role_menu` | 角色可操作的菜单与权限点清单。 |
| **角色部门关联表**| `sys_role_dept` | 角色的部门级数据权限范围。 |
| **角色继承表** | `sys_role_hierarchy` | 角色间的权限继承关系。 |
| **角色互斥表** | `sys_role_conflict` | 角色间的互斥约束。 |

### 2. `4.1.2-security_audit.sql` (安全审计日志)
**功能介绍**:
负责系统访问与业务行为的全链路审计记录。

**表清单**:
| 中文名称 | 英文名称 (Table Name) | 职责描述 |
| :--- | :--- | :--- |
| **登录记录表** | `sys_login_log` | 记录登录 IP、环境、状态等。 |
| **操作审计表** | `sys_oper_log` | 记录业务接口调用详情（参数、耗时等）。 |

### 3. `4.1.3-platform_config.sql` (平台基础配置)
**功能介绍**:
定义平台运行所需的静态配置表。

**表清单**:
| 中文名称 | 英文名称 (Table Name) | 职责描述 |
| :--- | :--- | :--- |
| **数据字典表** | `sys_dict` | 全局枚举值映射维护。 |
| **岗位管理表** | `sys_post` | 用户职能岗位定义。 |

---

## 🚀 4.2-初始化脚本模块

| 执行顺序 | 脚本名称 | 职能分类 |
| :--- | :--- | :--- |
| 1 | `4.2.1-infrastructure.sql` | 基础设施准备（库/用户/授权）。 |
| 2 | `4.2.2-RBAC_init.sql` | 权限体系初值。 |
| 3 | `4.2.3-sys_init.sql` | 系统默认参数/字典项。 |

---

## 📌 安装部署指引
1. **第一步**: 执行 `4.2.1-infrastructure.sql` 准备基础数据库环境。
2. **第二步**: 按顺序执行 `4.1.x-` 系列建表脚本定义库结构。
3. **第三步**: 按顺序执行 `4.2.x-` (从 4.2.2 开始) 初始化数据。
