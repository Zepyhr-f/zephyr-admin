# 系统管理功能详细设计

本文档详细描述了 Zephyr 系统管理模块的具体实现，包括用户、角色、菜单、部门等核心功能的 UI 展示与后端表关联。

## 1. 用户管理 (User Management)
管理系统的使用者账号及身份。

- **涉及表**：`sys_user` (核心), `sys_dept` (关联关联), `sys_user_role` (关联关联)
- **查询条件**：用户名、真实姓名、手机号、状态、所属部门。
- **列表展示字段**：
  - `real_name`: 真实姓名
  - `username`: 账号
  - `dept_name`: 部门名称 (关联 `sys_dept.dept_name`)
  - `phone`: 手机号
  - `status`: 状态 (正常/停用)
  - `create_time`: 创建时间
- **功能按钮**：
  - `新增`: 弹出对话框，选择部门（树选择器）、填写基本信息。
  - `编辑`: 修改用户信息，支持修改所属部门。
  - `删除`: 逻辑删除（修改 `del_flag = 1`）。
  - `重置密码`: 统一重置为系统默认密码。
  - `分配角色`: 弹窗由于勾选角色列表。

## 2. 角色管理 (Role Management)
定义系统的权限集合。

- **涉及表**：`sys_role` (核心), `sys_role_menu` (关联), `sys_role_dept` (关联)
- **列表展示字段**：
  - `role_name`: 角色名称
  - `role_code`: 权限字符 (如 `admin`)
  - `role_sort`: 显示顺序
  - `status`: 状态
  - `create_time`: 创建时间
- **功能按钮**：
  - `新增`: 填写信息，并为其分配初始菜单。
  - `编辑`: 修改基本信息。
  - `权限字符`: 修改 `role_code`。
  - `数据权限`: 配置 `data_scope`（全部、本部门等）。
  - `删除`: 逻辑删除。

## 3. 菜单管理 (Menu Management)
管理系统的路由、页面及按钮控制。

- **涉及表**：`sys_menu`
- **展示形式**：树形表格
- **核心字段**：
  - `menu_name`: 菜单名称
  - `icon`: 图标 (Antd 图标库)
  - `menu_type`: 类型 (M目录/C菜单/F按钮)
  - `perms`: 权限标识 (如 `sys:user:list`)
  - `path`: 路由地址 (仅 M/C)
  - `component`: 组件路径 (仅 C)
- **功能按钮**：
  - `新增下级`: 在当前节点下创建子节点。
  - `编辑`: 修改菜单属性。
  - `删除`: 若有下级子菜单则不允许直接删除。

## 4. 部门管理 (Dept Management)
定义组织架构。

- **涉及表**：`sys_dept`
- **展示形式**：树形结构
- **核心字段**：
  - `dept_name`: 部门名称
  - `order_num`: 排序
  - `status`: 状态
- **功能按钮**：
  - `新增/修改/删除`: 维护 tree 结构。

---

## 5. 交互规范
1. **删除操作**：必须由前端弹出 `Popconfirm` 或 `Modal` 确认提示，后端执行逻辑删除。
2. **状态切换**：列表中的状态列使用 `Switch` 组件，点击即触发接口更新。
3. **数据一致性**：删除用户时，应同步清理 `sys_user_role` 中该用户的记录。
