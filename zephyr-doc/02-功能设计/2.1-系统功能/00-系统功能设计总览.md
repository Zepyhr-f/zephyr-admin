# Zephyr 后台管理系统功能设计

## 一、 功能目录

### 1. 系统管理 (Core Admin)
核心 RBAC 权限体系，管理组织架构、人员与权限。
- **用户管理**：系统操作者的账号维护、分配角色。
- **部门管理**：配置系统组织机构（公司、部门、小组），树形展示。
- **岗位管理**：配置系统用户所属岗位。
- **菜单管理**：配置系统菜单、按钮、权限标识、路由等。
- **角色管理**：角色权限分配、设置角色按机构进行数据范围权限划分。

### 2. 安全审计 (Security & Audit)
保障系统安全，记录所有操作链路。具体实现方案请参考：[安全审计功能详细设计](./02-安全审计功能设计.md)。
- **登录日志**：记录系统用户的登录、登出行为及 IP、浏览器等信息。
- **操作日志**：记录所有业务接口的调用记录，包含请求参数、处理时长、执行结果。
- **在线用户**：实时展示当前系统中活跃的会话，支持强退操作。

### 3. 系统监控 (System Monitoring)
实时观察服务器状态与中间件运行指标。具体实现方案请参考：[系统监控功能详细设计](./03-系统监控功能设计.md)。
- **服务监控**：查看服务器 CPU、内存、磁盘、JVM 内存等状态。
- **缓存监控**：Redis 运行状态、命中率、内存占用、Key 数量。
- **数据源监控**：Druid 数据库连接池状态、慢 SQL 统计。
- **任务调度**：在线配置并管理 Cron 定时任务，支持日志追踪。

### 4. 基础设施 (Infrastructure)
系统稳定运行的基础组件管理。具体实现方案请参考：[基础设施功能详细设计](./04-基础设施功能设计.md)。
- **字典管理**：维护系统常用的固定数据（如：状态、性别、类别等）。
- **参数配置**：动态配置系统运行参数（如：文件存储路径、默认密码等）。
- **文件管理**：统一管理上传的静态资源（支持本地存储、OSS、Minio 等）。
- **通知公告**：发布系统公告，推送给所有用户。

### 5. 开发工具 (Developer Tools)
提升开发效率的自动化组件。具体实现方案请参考：[开发工具功能详细设计](./05-开发工具功能设计.md)。
- **代码生成**：根据数据库表结构，一键生成前后端 CRUD 代码及菜单 SQL。
- **接口文档**：集成 Swagger/Knife4j，实时在线预览、测试 API。
- **SQL 终端**：内置轻量级 Web SQL 执行面板（生产环境受限）。

## 二、 功能概览

| 模块名称 | 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **系统核心** | 用户/角色/菜单 | 实现颗粒度到按钮级的 RBAC | P0 |
| **系统核心** | 部门/岗位 | 实现基于组织架构的数据隔离 (Data Scope) | P0 |
| **安全守卫** | 登录/操作日志 | 满足合规性审计要求，全链路追踪异常 | P1 |
| **监控屏显** | 服务/缓存监控 | 实时预警，保障系统高可用性 | P1 |
| **快速开发** | 代码生成 | 减少 70% 的重复性工作，统一代码风格 | P2 |
| **文件中心** | 文件存储 | 抽象组件流，支持多协议存储切换 | P2 |

## 三、 特色设计建议

1.  **动态数据权限**：
    - 建议在 `Role` 表增加 `data_scope` 字段，支持“全部数据”、“本部门数据”、“本部门及以下数据”、“仅本人数据”四种模式。
2.  **多租户支持**：
    - 预留 `tenant_id` 字段，支持未来平滑升级为多租户平台。
3.  **灰度配置管理**：
    - 结合 Nacos，实现部分业务参数的“热修改”与“版本回滚”。
