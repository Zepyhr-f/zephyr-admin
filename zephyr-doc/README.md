## 📌 文档维护规则

1. **结构演进**：今后新增的文档应当逐渐贴合第一节中的 `0X-类目名称` 格式来建立目录或归档，以保持稳定有序的列表展示形式。
2. **格式要求**：所有文档统一使用 **Markdown** 格式编写，尽可能利用表格、列表或 Mermaid 等原生语法进行优雅排版。
3. **闭环更新机制**：当代码里修改了核心逻辑、数据库新增或删除了表字段、API 的路径及返回体发生了变更，**必须**同步更新本仓库下的相关设计思维导图与 MD 说明！
4. **自解释说明 (README)**：每个一级及二级模块目录下**必须**包含一个 `README.md` 文件。该文件用于概括本目录的功能职责、包含的文件清单以及后续修改的参考指引，以便 AI 助手和开发者能快速定位并执行任务。
5. **命名清晰化**：在归档时使用直白具体的中文命名结合 `-` 连接符（例如 `02-功能设计`），严禁建立无意义名称或存放临时垃圾文稿。
