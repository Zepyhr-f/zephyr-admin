# Zephyr 项目文档中心

本仓库是 Zephyr 项目的统一文档中心，目标是让两类读者都能快速定位正确信息：
- 人类开发者：按目录与主题快速找到入口、SSOT 和上下文。
- AI 助手：通过 README 的“关键词 -> 文件”索引稳定命中目标文档。

## 4. 文档治理基线（强制）

1. **以真实内容为准**：导航和 README 只能指向仓库中真实存在的目录和文件；未来规划不得伪装成已落地入口。
2. **先有 README 再有正文**：新增文档前，先更新所属目录的 `README.md`，补齐文件清单、关键词和维护说明。
3. **明确单一事实来源（SSOT）**：版本、端口、鉴权链路、数据库执行顺序等主题必须指定唯一主文档，其他文档只引用不重复发明口径。
4. **命名稳定**：文件名优先使用“编号-主题.md”，目录名优先使用“编号-类目”；本轮统一默认不做大规模物理重命名。
5. **变更闭环**：代码、接口、数据库、部署流程、权限模型发生变化时，必须同步更新对应文档。
6. **元信息最少集**：关键文档至少包含 `状态 / Owner / 最后更新 / 适用范围 / 关联文档`。

---

## 5. 统一维护动作

- 新增主题：先更新目录 `README.md`，再新增正文。
- 主题变更：先确认 SSOT 文档，再同步更新引用文档。
- 规划内容：统一收敛到“规划中 / 待建设”区块，禁止创建失效链接。
- 发现死链：在合入前修复，README 中出现死链视为文档质量问题。
