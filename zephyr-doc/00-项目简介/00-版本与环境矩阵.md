## 1. 开发机工具链（必选）

| 分类 | 工具 | 版本/范围 | 说明 |
|---|---|---|---|
| 后端 | JDK | **21 LTS** | 统一以 JDK 21 作为开发/CI 基线（如需兼容更低版本，请在此处声明） |
| 后端 | Maven | 3.9+ | 建议与 CI 保持一致 |
| 前端 | Node.js | **22 LTS** | 以项目根目录 `.nvmrc` 与 `package.json#engines` 为准 |
| 前端 | pnpm | 10.x | 以 `package.json#packageManager` 为准（示例：`pnpm@10.8.0`） |
| 基础 | Git | 2.40+（建议） | 需支持现代 Gitflow/Hook 工具链 |
| 容器 | Docker / OrbStack | 最新稳定版 | macOS 可选 OrbStack 替代 Docker Desktop |

> [!TIP]
> 建议在仓库根目录补充：`.nvmrc`、`package.json#engines`、以及（后端）`.java-version` 或 `maven-toolchains.xml`，进一步“用代码锁死版本”。

---

## 2. 运行依赖（本地/测试环境）

| 组件 | 推荐版本 | 用途 |
|---|---|---|
| MySQL | 8.0+ | 主数据库 |
| Redis | 7.2+ | Token/权限缓存/会话 |
| Nacos | 2.x | 注册中心与配置中心 |
| MinIO | 最新稳定版 | 本地对象存储（文件上传） |

> [!NOTE]
> 若线上环境采用 OceanBase、云 OSS 等，请在此补充“生产等价替代方案”和差异点（兼容性、SQL 方言、权限策略、限流等）。

## 3. 常用端口约定（建议）

| 模块 | 默认端口 | 说明 |
|---|---:|---|
| 前端 Vite DevServer | 5173（或按项目配置） | 以终端启动输出为准；如改为 8000/3000，请在此登记 |
| Gateway | （待补充） | 外部 API 统一入口 |
| Nacos | 8848 | 控制台/注册发现 |
| MySQL | 3306 | 数据库 |
| Redis | 6379 | 缓存 |
| MinIO | 9000 / 9001 | API/Console |

---

## 4. 版本变更流程（强制）

1. 任何人提出版本变更（例如 Node 22 → 24、Spring Boot 升级）必须先更新本文档并说明原因。
2. 同步更新受影响的文档（快速开始/环境搭建/规范）与脚本（`.nvmrc`、`packageManager`、CI 配置）。
3. 由 Owner/Reviewer 评审通过后，才允许合入主分支。
