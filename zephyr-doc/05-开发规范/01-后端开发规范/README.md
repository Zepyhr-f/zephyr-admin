## 阅读建议
- 新同学先看《项目技术栈与模块结构》建立后端开发上下文。
- 落地编码前，再看《编程质量与 Clean Code 规范》和《Java 与 Spring 最佳实践》。
- 涉及提交动作时，补看《Git 工作流与提交规范》。

## 维护规则
- 本目录只维护后端开发规范，不承载具体功能方案。
- 与全局发版流程冲突时，以通用规范目录中的发版 SOP 为准。
- 版本相关描述应尽量引用版本矩阵，避免在本目录复制版本口径。
