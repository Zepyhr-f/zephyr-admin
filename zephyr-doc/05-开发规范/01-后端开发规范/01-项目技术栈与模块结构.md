# 项目技术栈与模块结构规范

## 技术栈

### 核心框架与环境
*   **Java 17 (LTS)**: 项目采用的长期支持版 Java，利用其 Seal Classes、Record 等新特性。
*   **Spring Boot 3.3.4**: 核心应用框架，基于 Spring Framework 6。
*   **Spring Cloud 2023.0.3**: 微服务治理方案。
*   **Spring Cloud Alibaba 2022.0.0.0**: 结合国内生态的微服务套件（Nacos, Sentinel, Seata）。
*   **Maven**: 模块化依赖管理与构建工具。

### 数据持久化
*   **MySQL 8.0.33**: 核心关系型数据库。
*   **MyBatis-Plus 3.5.6**: 基于 MyBatis 的增强工具，提供高效的 CRUD 开发与分压插件。
*   **Redis**: 分布式缓存与共享会话存储。

### 开发效率工具
*   **Lombok**: 通过注解消除冗长的 Java 代码（Getter, Setter, Slf4j 等）。
*   **MapStruct**: 高性能的对象属性映射工具（DTO <=> Entity）。
*   **Hutool**: Java 常用工具类库。

## 模块结构规范

Zephyr 后端采用 Maven 多模块架构，遵循“核心功能内聚、业务功能解耦”的原则：

```
zephyr-server/
├── zephyr-base/                # 基础基础设施模块
│   ├── zephyr-starter-core/    # 核心工具类与通用配置
│   ├── zephyr-starter-mp/      # MyBatis-Plus 增强配置（BaseEntity, 自动填充）
│   ├── zephyr-starter-redis/   # Redis 与序列化配置
│   └── zephyr-starter-security/# 基于 Spring Security 的安全体系
├── zephyr-common/              # 公共组件与常量（不含业务逻辑）
├── zephyr-gateway/             # 基于 Spring Cloud Gateway 的统一网关
└── zephyr-biz/                 # 业务微服务实现
    ├── zephyr-biz-system/      # 系统管理模块 (用户、角色、菜单、部门)
    └── zephyr-biz-gen/         # 代码生成工具模块
```

### 业务微服务内部结构 (Standard Layering)
每个 `biz` 微服务通常遵循以下包结构：

```
com.zephyr.[module]/             # 根包名
├── controller/                  # RESTful API 控制层（仅处理请求路由与验证）
├── service/                     # 业务接口层
│   └── impl/                    # 业务实现层（包含事务管理与复杂业务逻辑）
├── mapper/                      # 数据持久化接口层 (DAO)
├── entity/                      # 数据库实体类（对应数据库表）
├── domain/                      # 领域对象
│   ├── dto/                     # Data Transfer Object（接收/传输数据）
│   └── vo/                      # View Object（返回给前端展示的对象）
└── util/                        # 模块内部工具类
```

> [!IMPORTANT]
> **绝对禁止**在 `Controller` 层编写业务逻辑。Controller 只应负责请求参数的基本验证（使用 `@Valid`）和调用 Service。事务控制（`@Transactional`）应统一加在 `Service` 实现类上。
