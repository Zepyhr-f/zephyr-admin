# 数据库建表规范

> **适用范围**：Zephyr 项目所有微服务数据库表
> **强制级别**：所有新建表必须严格遵循本规范，存量表逐步对齐

---

## 一、基础字段规范（必须字段）

Zephyr 项目所有业务表都必须继承统一的基础字段集合，对应后端框架基类 `BaseEntity`。

### 1.1 基础字段定义

| 字段名        | 数据类型              | 是否必须 | 默认值                                          | 说明                                   |
| ------------- | --------------------- | -------- | ----------------------------------------------- | -------------------------------------- |
| `id`          | `BIGINT` NOT NULL     | ✅ 是    | -                                               | 主键 ID，雪花算法或自增                |
| `tenant_code` | `VARCHAR(12)`         | ✅ 是    | `'000000'`                                      | 租户编码，用于多租户数据隔离           |
| `create_user` | `VARCHAR(64)`         | ✅ 是    | NULL                                            | 创建人编码                             |
| `create_time` | `DATETIME`            | ✅ 是    | `CURRENT_TIMESTAMP`                             | 创建时间                               |
| `update_user` | `VARCHAR(64)`         | ✅ 是    | NULL                                            | 最后更新人编码                         |
| `update_time` | `DATETIME`            | ✅ 是    | `CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 最后更新时间                           |
| `if_deleted`  | `TINYINT(1)` NOT NULL | ✅ 是    | `0`                                             | 逻辑删除标志：`0` = 正常，`1` = 已删除 |

> [!IMPORTANT]
> **`if_deleted` 字段是逻辑删除的唯一标准方式**，严禁使用物理删除（`DELETE FROM`）操作业务数据。MyBatis-Plus 框架已在 `BaseEntity` 中通过 `@TableLogic` 自动拦截。
> `id` 字段仅做数据唯一标识，涉及业务唯一性需额外创建 `xxx_code` 等字段。

### 1.2 对应后端 BaseEntity 类

```java
// 路径: zephyr-base/zephyr-starter/zephyr-starter-mp/src/main/java/com/zephyr/mp/base/BaseEntity.java
public class BaseEntity implements Serializable {

    @TableId(value = "id", type = IdType.ASSIGN_ID)
    private Long id;

    private String tenantCode;  // 租户编码

    private String createUser;    // 创建人编码
    private Date createTime;      // 创建时间
    private String updateUser;    // 最后更新人编码
    private Date updateTime;      // 最后更新时间

    @TableField("if_deleted")
    @TableLogic(value = "0", delval = "1")
    private Integer ifDeleted;
}
```

## 二、建表 SQL 标准模板

### 2.1 通用业务表模板

```sql
CREATE TABLE `zephyr_业务模块_表名` (
  -- ① 主键
  `id`          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键ID',

  -- ② 业务核心字段
  `xxx_code`    VARCHAR(64)  NOT NULL COMMENT '业务编码 (建议作为唯一标识)',
  -- ...

  -- ③ 基础审计与租户字段
  `tenant_code` VARCHAR(12)  DEFAULT '000000' COMMENT '租户编码',
  `create_user` VARCHAR(64)  DEFAULT NULL COMMENT '创建人编码',
  `create_time` DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_user` VARCHAR(64)  DEFAULT NULL COMMENT '更新人编码',
  `update_time` DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `if_deleted`  TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '删除标识（0=正常 1=已删除）',

  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_xxx_code_del` (`xxx_code`, `if_deleted`, `tenant_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='【填写表注释】';
```

### 2.2 仅含关联关系的中间表（无需继承 BaseEntity）

中间表（如 `sys_user_role`、`sys_role_menu`）通常只存储两个外键，**不需要审计字段**：

```sql
CREATE TABLE `zephyr_sys_user_role` (
  `user_code` VARCHAR(64) NOT NULL COMMENT '用户编码',
  `role_code` VARCHAR(64) NOT NULL COMMENT '角色编码',
  PRIMARY KEY (`user_code`, `role_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户和角色关联表';
```

## 三、字段命名规范

### 3.1 通用命名约定

| 规则              | 说明                                                | 示例                                         |
| ----------------- | --------------------------------------------------- | -------------------------------------------- |
| 全部小写 + 下划线 | 使用 snake_case 风格                                | `user_name`、`dept_id`                       |
| 布尔/状态类字段   | 用 `status`（业务状态）或 `0/1` 枚举型 `TINYINT(1)` | `status TINYINT(1)`                          |
| 外键字段          | 以关联表名的单数形式加 `_id` 结尾                   | `role_id`、`dept_id`、`menu_id`              |
| 时间字段          | 统一用 `_time` 结尾，类型为 `DATETIME`              | `create_time`、`expire_time`                 |
| 金额字段          | 使用 `DECIMAL(18, 2)` 类型，单位说明写在注释        | `amount DECIMAL(18, 2) COMMENT '金额（元）'` |
| 文本类字段        | 短文本用 `VARCHAR`，长文本用 `TEXT`                 | `remark VARCHAR(500)`、`content TEXT`        |

### 3.2 禁止事项

- ❌ 禁止使用 `MySQL` 关键字作为字段名（如 `key`、`value`、`desc`、`order`）
- ❌ 禁止使用 `is_xxx` 形式命名（与 JavaBean 规范冲突），改用 `xxx_flag` 或 `status`（逻辑删除统一用 `if_deleted`）
- ❌ 禁止使用 `FLOAT` / `DOUBLE` 存储金额，必须用 `DECIMAL`
- ❌ 禁止在 `VARCHAR` 字段不加长度限制

## 四、索引规范

### 4.1 索引命名

| 类型     | 前缀            | 示例            |
| -------- | --------------- | --------------- |
| 普通索引 | `idx_`          | `idx_username`  |
| 唯一索引 | `uk_`           | `uk_role_code`  |
| 联合索引 | `idx_` + 多字段 | `idx_user_role` |

### 4.2 索引创建原则

- `PRIMARY KEY` 必须是 `id`，建议使用 `BIGINT`。
- 业务唯一标识（如 `code` 类字段）必须建立唯一索引，且**必须包含 `if_deleted` 和 `tenant_code`** 以支持逻辑删除和多租户下的唯一性。
- 唯一索引命名：`uk_字段名_del`。
- 查询频率高的字段加普通索引 `idx_字段名`。
- 避免在低基数字段（如 `status`、`if_deleted`）上创建单独索引。

---

## 五、字符集与存储引擎规范

- **字符集**：统一使用 `utf8mb4`，支持 Emoji 及四字节字符
- **排序规则**：`utf8mb4_general_ci`（不指定时使用数据库默认值）
- **存储引擎**：统一使用 `InnoDB`，支持事务与外键

## 六、完整建表示例

以下是一个标准的业务表建表示例（以部门表 `sys_dept` 为参考）：

```sql
CREATE TABLE `zephyr_sys_dept` (
  `id`          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `dept_code`   VARCHAR(64)  NOT NULL COMMENT '部门编码',
  `parent_code` VARCHAR(64)  DEFAULT '0' COMMENT '父部门编码',
  `dept_name`   VARCHAR(50)  NOT NULL COMMENT '部门名称',
  `order_num`   INT          DEFAULT 0 COMMENT '显示顺序',
  `status`      TINYINT(1)   DEFAULT 1 COMMENT '部门状态（1=正常 0=停用）',

  -- 基础审计与租户字段
  `tenant_code` VARCHAR(12)  DEFAULT '000000' COMMENT '租户ID',
  `create_user` VARCHAR(64)  DEFAULT NULL COMMENT '创建人编码',
  `create_time` DATETIME     DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_user` VARCHAR(64)  DEFAULT NULL COMMENT '更新人编码',
  `update_time` DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `if_deleted`  TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '删除标识（0=正常 1=已删除）',

  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_dept_code_del` (`dept_code`, `if_deleted`, `tenant_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='部门表';
```

## 七、后端 Entity 类对应规范

每张业务表都必须有对应的 Java Entity 类，且须继承 `BaseEntity`：

```java
@Data
@EqualsAndHashCode(callSuper = true)
@TableName("zephyr_sys_dept")
public class Dept extends BaseEntity {

    private String deptCode;
    private String parentCode;
    private String deptName;
    private Integer orderNum;
    private Integer status;
}
```

> [!TIP]
> `BaseEntity` 已通过 MyBatis-Plus 自动填充机制处理 `create_user`、`create_time`、`update_user`、`update_time`，无需在代码中手动赋值。
