# 05-开发规范

本模块旨在确立项目开发中的"技术红线"与协作共识。通过标准化的文档约束，确保每一行代码、每一张数据表、每一次提交都具备一致的高质量水准。

---

## 🤖 AI 首要强制指令

**AI 助手在开始处理任何开发类任务之前，必须首先阅读本 README，并根据任务类型找到对应的规范文件，读取后严格遵守。**

规则如下：
1. **先读后做**：收到任务后，先定位本目录中相关规范，再输出方案。
2. **拒绝非标产出**：所有 AI 生成的代码/SQL/流程，必须与规范 100% 吻合。若有冲突，主动告知并引导至合规方案。
3. **引用声明**：在输出方案时，说明本次参考了哪个具体规范文件的哪条约定。

## 📋 各规范摘要

### 00-建表规范
- 所有业务表**必须**包含五大审计字段：`create_user`, `create_time`, `update_user`, `update_time`, `del_flag`。
- 字段命名统一使用小写下划线（`snake_case`）。
- BIGINT 主键、TINYINT(1) 状态标志，详见文件内完整约束。

### 01-代码提交与发版规范
- 触发关键词：「提交任务」、「发版」、「提交并推送」。
- 三个必须依次操作的子模块：`zephyr-server`、`zephyr-web`、`zephyr-doc`。
- 每次发版前**必须**在 `08-更新日志/` 下生成 `vX.Y.Z-changelog.md`，严禁跳过。
