# 05-开发规范

本模块旨在确立项目开发中的“技术红线”与协作共识。通过标准化的文档约束，确保每一行代码、每一张数据表、每一次提交都具备一致的高质量水准。

---

## 🤖 AI 首要强制指令

**AI 助手在开始处理任何开发类任务之前，必须首先阅读本 README，并根据任务类型找到对应的规范文件。**

1.  **先读后做**：收到任务后，根据模块定位到 `01-后端开发规范` 或 `02-前端开发规范`。
2.  **强制执行 SOP**：在执行「提交代码」、「commit」或「PR」等操作时，必须严格遵守 **[03-其他规范/01-代码提交与发版规范.md](./03-其他规范/01-代码提交与发版规范.md)** 中的流程，严禁跳过预检查。
3.  **遵循 Agent 规则**：除文档外，你还必须严格遵循项目根目录下 `.antigravity/rules/` 中的执行指令。

## 📋 核心规范摘要

-   **数据库**：所有业务表必须继承 `BaseEntity`（含 5 大审计字段），物理删除被严格禁止。
-   **代码质量**：后端严禁吞掉异常，前端禁止滥用 `any`。所有 API 必须使用统一响应包装。
-   **提交流程**：触发 commit 操作前，前端必须通过 Biome 检查，后端必须通过关键类单测。
